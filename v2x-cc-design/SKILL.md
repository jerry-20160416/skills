# V2X **项目设计文档生成器

本 Skill 用于按照***项目的标准格式生成模块详细设计文档。

## 设计文档标准结构

每份设计文档必须包含以下章节：

```
# <模块名称> 详细设计

## 文档信息
| 项目 | 内容 |
|------|------|
| 文档名称 | <模块>详细设计 |
| 版本 | V1.0 |
| 日期 | YYYY-MM-DD |
| 适用范围 | ***项目 - <子系统> |

## 目录
1. [需求分析](#1-需求分析)
2. [总体架构](#2-总体架构)
3. [数据模型](#3-数据模型)
4. [核心算法](#4-核心算法)
5. [关键流程](#5-关键流程)
6. [性能优化](#6-性能优化)
7. [接口设计](#7-接口设计)
```

---

## 各章节内容规范

### 1. 需求分析

必须包含三类表格：

**数据规模表：**
```markdown
| 数据类型 | 数量 | 说明 |
|----------|------|------|
| ... | ... | ... |
```

**场景需求表：**
```markdown
| 场景 | 功能需求 | 性能要求 |
|------|----------|----------|
| ... | ... | ... |
```

**性能指标表：**
```markdown
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 响应延迟 | < X ms | ... |
| 吞吐量 | > X 次/秒 | ... |
| 内存占用 | < X MB | ... |
```

## 使用方式

当用户需要撰写设计文档时：

1. **询问模块信息**：模块名称、所属子系统、核心功能
2. **生成文档框架**：按标准结构创建文档骨架
3. **填充内容**：根据用户提供的需求、场景、性能指标填充各章节
4. **补充技术细节**：结合领域知识补充数据模型、算法实现、接口定义

## 输出要求

- 文档保存路径：`docs/<module-name>-design.md`
- 使用 Markdown 格式
- 图表使用 ASCII 艺术（便于版本控制）
- 代码使用 Java 语言（项目标准）
- 表格使用标准 Markdown 表格格式
- 所有中文内容
