# 阿里巴巴 Java 开发手册（黄山版）

## 何时激活此 Skill

- 编写或审查 Java 代码时
- 进行代码规范检查、Code Review
- 制定项目编码规范、团队开发标准
- 处理异常日志、单元测试、数据库设计
- 进行安全审计、工程结构规划
- 设计系统架构、模块划分
- 询问命名规范、代码格式、注释规范等具体问题

## 手册维度

本手册以 Java 开发者为中心视角，划分为 **七个维度**：

### 一、编程规约
- **命名风格**：类名、方法名、变量名、常量名等命名规范
- **常量定义**：常量的定义、使用、管理规范
- **代码格式**：代码缩进、空格、换行、大括号等格式规范
- **OOP 规约**：面向对象设计原则与规约
- **日期时间**：日期时间的处理规范
- **集合处理**：集合类的使用规范
- **并发处理**：多线程、并发编程规范
- **控制语句**：条件语句、循环语句等控制流程规范
- **注释规约**：代码注释的编写规范
- **前后端规约**：前后端交互的数据格式、命名等规范
- **其他**：其他编程相关规约

### 二、异常日志
- **错误码**：错误码的定义、使用、管理规范
- **异常处理**：异常捕获、抛出、处理的规范
- **日志规约**：日志级别、格式、输出的规范

### 三、单元测试
- 单元测试编写规范
- 测试覆盖率要求
- 测试用例设计原则

### 四、安全规约
- 代码安全编写规范
- 权限控制、鉴权规范
- 敏感数据处理规范

### 五、MySQL 数据库
- **建表规约**：表设计、字段类型、命名规范
- **索引规约**：索引设计、使用规范
- **SQL 语句**：SQL 编写规范
- **ORM 映射**：ORM 框架使用规范

### 六、工程结构
- **应用分层**：项目分层架构规范
- **二方库依赖**：依赖管理规范
- **服务器**：服务器部署、配置规范

### 七、设计规约
- 系统设计原则
- 模块划分规范
- 接口设计规范

## 规约等级说明

依据约束力强弱及故障敏感性，规约分为三大类：

| 等级 | 标识 | 说明 |
|------|------|------|
| **强制** | 【强制】 | 必须遵守，违反将导致严重问题 |
| **推荐** | 【推荐】 | 建议遵守，有助于提升代码质量 |
| **参考** | 【参考】 | 可选遵守，根据实际情况灵活处理 |

## 使用方式

### 1. 查询具体规约
直接询问具体问题，例如：
- "命名规范是什么？"
- "如何处理异常？"
- "数据库建表有哪些要求？"
- "日志该怎么写？"

### 2. 代码审查
在进行 Code Review 时，对照手册规约检查代码：
- 检查命名是否符合规范
- 检查异常处理是否正确
- 检查日志输出是否合规
- 检查 SQL 语句是否规范

### 3. 项目规范制定
根据手册内容，制定符合项目实际情况的编码规范文档。
